# Конспект лекции №18: Инструменты для работы с API

---

## 1. Задачи аналитика при работе с API

Аналитик выполняет следующие задачи:

| Задача | Описание |
|--------|----------|
| **Проектирование** | Создание API для backend-разработчиков |
| **Документирование** | Описание API в удобном для всех формате |
| **Анализ** | Изучение чужих API для интеграции |
| **Тестирование** | Проверка реализации API после разработки |

---

## 2. Postman — универсальный инструмент

**Postman** — это программа для работы с API (бесплатная, open-source).

### Возможности:

| Функция | Описание |
|---------|----------|
| **Коллекции** | Группировка запросов по проектам/сервисам |
| **Переменные окружения** | Подстановка значений (например, токенов) в нескольких запросах |
| **Импорт** | Можно импортировать готовую коллекцию или cURL-команду |
| **История** | Хранит все выполненные запросы |
| **Скрипты** | Автоматизация (например, сохранить токен после авторизации) |
| **Тестирование** | Автоматические проверки ответов (сравнение с ожидаемыми данными) |
| **Примеры (Examples)** | Фиксация разных состояний запроса (200, 400, 500) |
| **Визуализация** | Отображение ответа в виде JSON, XML, таблицы или HTML |
| **Документирование** | Автоматическая документация из коллекции |

### Структура интерфейса:

```
[Вкладка запроса] → [HTTP-метод] → [URL] → [Send]
                     [Params] [Auth] [Headers] [Body] [Scripts]
                     ↓
                     [Response: Status, Body]
```

**Важно:** Postman позволяет **делиться коллекциями** → разработчики и тестировщики используют одинаковые контракты.

## 3. Swagger (OpenAPI) — документация API

**Swagger** — это инструмент для создания интерактивной документации API.

### Особенности:

- Работает только с **REST API**
- **Интерактивность:** можно выполнять запросы прямо из документации
- **Автогенерация:** разработчики могут генерировать документацию из кода
- **Версионирование:** поддержка нескольких версий API
- **Переиспользуемые компоненты:** модели данных, которые можно использовать в разных запросах

### Как выглядит:

```
[Название API]
  └── [Группа запросов]
       └── [Метод] [Путь]
            ├── Параметры (Path, Query, Body)
            ├── Ответ (структура, примеры)
            └── Кнопка "Try it out"
```

### Плюсы:
- Удобство для разработчиков и аналитиков
- Наглядность и простота использования
- Можно писать вручную в специальном формате (YAML/JSON) через онлайн-редактор

### Минусы:
- Сложные структуры требуют дополнительных настроек
- Требует внимательности при ручном написании

> **На практике:** большинство разработчиков используют **автогенерацию** (плагины для фреймворков). Аналитики чаще проверяют работу, чем пишут Swagger вручную.

## 4. Инструменты разработчика в браузере (DevTools)

**DevTools** — встроенная панель во всех браузерах (открывается по `F12`).

### Основные вкладки и их использование:

| Вкладка | Что показывает | Использование для аналитика |
|---------|----------------|----------------------------|
| **Elements** | HTML-разметка страницы | Просмотр структуры, адаптивность, поиск элементов |
| **Console** | Логи, выполнение JS-команд | Отладка, запуск скриптов |
| **Sources** | Исходные файлы (HTML, CSS, JS) | Просмотр кода |
| **Network** | HTTP-запросы и ответы | **Анализ запросов, проверка API** |
| **Application** | Cookies, Local Storage, Session Storage | Получение токенов, проверка сохранённых данных |

### Вкладка Network (самая полезная для аналитика):

```
[Список запросов] → [Выбор запроса]
                      ├── Headers (метод, URL, заголовки, куки)
                      ├── Payload (тело запроса)
                      ├── Preview (ответ в структурированном виде)
                      ├── Response (сырой ответ)
                      └── Timing (время выполнения)
```

### Полезные функции:
- **Инспектор элементов** (стрелочка) — клик на элемент → открытие его в HTML-коде
- **Адаптивность** — переключение отображения на мобильные устройства
- **Очистка лога** — кнопка "Clear" для фокуса на конкретном запросе
- **Фильтр** — поиск по названиям запросов

### Где взять токен для Postman?
1. Открыть DevTools → Network
2. Выполнить действие на сайте
3. Найти запрос с нужным заголовком
4. Посмотреть `Authorization: Bearer <токен>`

---

## 5. Итоговые тезисы (шпаргалка)

1. **Postman** — универсальный инструмент для работы с API (тестирование, анализ, документирование)
2. **Коллекции** — группировка запросов, их можно передавать команде
3. **Swagger** — интерактивная документация с возможностью выполнять запросы
4. DevTools → **Network** — главный инструмент для анализа существующих API
5. В Network можно посмотреть **метод, URL, заголовки, тело, ответ, время выполнения**
6. **Cookies** и **Local Storage** — места хранения токенов на фронтенде
7. Все три инструмента **взаимодополняют** друг друга в работе аналитика
