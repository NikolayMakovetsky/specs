# Конспект лекции №10: Базовый синтаксис SQL

---

## 1. Зачем системному аналитику знать SQL?

1. **Усиливает логическое мышление** и помогает понимать язык разработчиков
2. Позволяет разбираться в **коде**, когда документация неактуальна
3. Помогает **формулировать нефункциональные требования** с учётом узких мест
4. Даёт возможность **оценивать реализуемость** требований на уровне БД

---

## 2. Что такое SQL?

**SQL (Structured Query Language)** — язык структурированных запросов, который позволяет:
- Хранить данные
- Извлекать данные
- Обрабатывать и манипулировать данными

---

## 3. Операторы DDL (Data Definition Language)

### CREATE TABLE — создание таблицы

```sql
CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    PRIMARY KEY (column1)
);
```

### DROP TABLE — удаление таблицы

```sql
DROP TABLE table_name;
```

> **Важно:** DROP TABLE удаляет и структуру, и данные. DROP vs DELETE — большая разница.

---

## 4. Операторы DML (Data Manipulation Language)

### INSERT — добавление записей

```sql
INSERT INTO users (user_id, user_name, age, status)
VALUES (1, 'Анна', 25, 'active'),
       (2, 'Игорь', 30, 'active');
```

**Правила:**
- Если не перечислять колонки, значения должны идти в порядке объявления
- Строковые значения заключаются в кавычки `''`
- Можно вставлять несколько строк в одном INSERT

---

### SELECT — выборка данных

```sql
SELECT user_id, user_name, age
FROM users;
```

```sql
SELECT * FROM users;  -- все колонки
```

**Сортировка:**
```sql
SELECT *
FROM users
ORDER BY city ASC, age DESC;  -- ASC по умолчанию
```

---

### WHERE — фильтрация данных

```sql
SELECT *
FROM users
WHERE status = 'active' AND age > 26;
```

**Операторы в WHERE:**
- `=`, `>`, `<`, `>=`, `<=`, `<>` (не равно)
- `LIKE` — поиск по шаблону
- `AND`, `OR` — логические условия

---

### UPDATE — обновление записей

```sql
UPDATE users
SET age = 30
WHERE user_name = 'Игорь';
```

> **Всегда используйте WHERE в UPDATE, если не нужно обновить все записи!**

---

### DELETE — удаление записей

```sql
DELETE FROM users
WHERE status = 'inactive';
```

> **Осторожно:** DELETE без WHERE удаляет **все** записи из таблицы (таблица остаётся).

---

## 5. Агрегатные функции

| Функция | Описание |
|---------|----------|
| `AVG()` | Среднее значение |
| `SUM()` | Сумма |
| `MIN()` | Минимальное значение |
| `MAX()` | Максимальное значение |
| `COUNT()` | Количество записей |

**Пример:**
```sql
SELECT COUNT(*) AS records_count
FROM users;  -- вернёт количество записей
```

---

## 6. Псевдонимы (Aliases)

**Псевдоним** — временное имя для таблицы или колонки, улучшающее читаемость.

```sql
SELECT COUNT(*) AS total_users
FROM users AS u
WHERE u.status = 'active';
```

---

## 7. GROUP BY — группировка

```sql
SELECT city, COUNT(*) AS user_count
FROM users
GROUP BY city
ORDER BY city;
```

**Правила:**
- GROUP BY всегда идёт **после** WHERE
- GROUP BY всегда идёт **перед** ORDER BY
- В SELECT можно использовать только поля из GROUP BY или агрегатные функции

---

## 8. HAVING — фильтрация групп

**HAVING** фильтрует результаты агрегации и всегда идёт **после** GROUP BY.

```sql
SELECT city, COUNT(*) AS user_count
FROM users
GROUP BY city
HAVING COUNT(*) > 1;
```

### Отличие WHERE от HAVING:

| WHERE | HAVING |
|-------|--------|
| Фильтрует **строки до группировки** | Фильтрует **группы после группировки** |
| Не может использовать агрегатные функции | Может использовать агрегатные функции |
| Идёт **до** GROUP BY | Идёт **после** GROUP BY |

---

## 9. JOIN — соединение таблиц

**JOIN** используется для объединения данных из нескольких связанных таблиц.

### Типы JOIN:

| Тип | Описание |
|-----|----------|
| **INNER JOIN** | Только совпадающие записи из обеих таблиц |
| **LEFT JOIN** | Все записи из левой таблицы + совпадающие из правой |
| **RIGHT JOIN** | Все записи из правой таблицы + совпадающие из левой |
| **FULL JOIN** | Все записи из обеих таблиц |
| **CROSS JOIN** | Декартово произведение |

**Общая конструкция:**
```sql
SELECT t1.col1, t2.col2
FROM table1 t1
[INNER|LEFT|RIGHT|FULL] JOIN table2 t2
ON t1.key = t2.key
[WHERE condition];
```

---

### Пример: INNER JOIN

```sql
SELECT u.user_id, u.user_name, d.department_name
FROM users u
INNER JOIN departments d ON u.department_id = d.department_id;
```

---

### Пример: LEFT JOIN

```sql
SELECT u.user_id, u.user_name, d.department_name
FROM users u
LEFT JOIN departments d ON u.department_id = d.department_id;
```

> **Левая таблица** — та, что указана перед LEFT JOIN. Все её записи будут в результате.

---

## 10. Подзапросы (Subqueries)

**Подзапрос** — это запрос внутри другого запроса (заключён в круглые скобки).

### Типы подзапросов:

| Тип | Возвращает |
|-----|------------|
| **Скалярный** | Одно значение (одна строка, одна колонка) |
| **Табличный** | Множество значений (несколько строк/колонок) |

### По способу выполнения:

| Тип | Описание |
|-----|----------|
| **Простой** | Выполняется независимо от внешнего запроса |
| **Связанный (коррелированный)** | Зависит от внешнего запроса (выполняется для каждой строки) |

---

### Пример: скалярный подзапрос

```sql
SELECT part_name
FROM parts
WHERE price > (SELECT price FROM parts WHERE part_name = 'Bolt');
```

---

### Пример: табличный подзапрос с IN

```sql
SELECT supplier_name
FROM suppliers s
WHERE s.supplier_id IN (
    SELECT supplier_id
    FROM supplies
);
```

---

### Пример: связанный подзапрос с EXISTS

```sql
SELECT supplier_name
FROM suppliers s
WHERE EXISTS (
    SELECT 1
    FROM supplies sp
    WHERE sp.supplier_id = s.supplier_id
);
```

> **Важно:** EXISTS проверяет, возвращает ли подзапрос хотя бы одну строку.

### Операторы для подзапросов:

| Оператор | Описание |
|----------|----------|
| `IN` | Принадлежит множеству |
| `ANY`/`SOME` | Хотя бы одно значение удовлетворяет условию |
| `ALL` | Все значения удовлетворяют условию |
| `EXISTS` | Существует хотя бы одна строка |

---

## 11. Плюсы и минусы подзапросов

### Плюсы:
- Выполняются без создания временных структур
- Гибкость — можно использовать с SELECT, INSERT, UPDATE, DELETE
- Удобны для сложных фильтров и агрегаций

### Минусы:
- Влияют на производительность (могут выполняться многократно)
- Неэффективны с большими объёмами данных

> **Для больших данных лучше использовать JOIN вместо подзапросов.**

---

## 12. Быстрая памятка (шпаргалка)

### Порядок выполнения SQL-запроса:
1. `FROM` (определяем таблицы)
2. `JOIN ... ON` (соединяем)
3. `WHERE` (фильтруем строки)
4. `GROUP BY` (группируем)
5. `HAVING` (фильтруем группы)
6. `SELECT` (выбираем колонки)
7. `ORDER BY` (сортируем)
8. `LIMIT` (ограничиваем результат)

### Базовый шаблон запроса:
```sql
SELECT [колонки, агрегации]
FROM [таблицы]
[JOIN ... ON ...]
[WHERE условия]
[GROUP BY поля]
[HAVING условия для групп]
[ORDER BY поля [ASC|DESC]]
```
