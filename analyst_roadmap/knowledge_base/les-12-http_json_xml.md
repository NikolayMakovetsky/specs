# Конспект лекции №12: Основы протокола HTTP, форматы данных (JSON, XML)

---

## 1. Что такое HTTP?

**HTTP (HyperText Transfer Protocol)** — протокол передачи гипертекста.

### Основная концепция:
- Клиент отправляет **запрос** на сервер
- Сервер обрабатывает запрос и возвращает **ответ**

### Архитектура: клиент-серверная
- **Клиент** — устройство/приложение пользователя (браузер, мобильное приложение)
- **Сервер** — обрабатывает запросы и возвращает ответы

---

## 2. Структура HTTP-запроса

| Компонент | Описание | Пример |
|-----------|----------|--------|
| **Метод** | Действие, которое нужно выполнить | GET, POST, PUT, DELETE |
| **Путь (URI)** | Ресурс, к которому обращаемся | `/users`, `/applications/123` |
| **Протокол и версия** | Версия HTTP | `HTTP/1.1` |
| **Заголовки (Headers)** | Мета-информация о запросе (необязательно) | `Content-Type: application/json` |
| **Тело (Body)** | Данные запроса (не для GET) | JSON, XML |

### Пример HTTP-запроса:
```
POST /users HTTP/1.1
Host: api.example.com
Content-Type: application/json
Authorization: Bearer token123

{
  "name": "Иван",
  "age": 30
}
```

---

## 3. Структура HTTP-ответа

| Компонент | Описание | Пример |
|-----------|----------|--------|
| **Протокол и версия** | Версия HTTP | `HTTP/1.1` |
| **Статус-код** | Код результата | `200`, `404`, `500` |
| **Заголовки** | Мета-информация об ответе | `Content-Type: application/json` |
| **Тело (Body)** | Данные ответа (необязательно) | JSON с результатом |

### Пример HTTP-ответа:
```
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 123,
  "name": "Иван",
  "created": "2024-01-01"
}
```

---

## 4. Методы HTTP

| Метод | Описание | Есть ли тело? |
|-------|----------|---------------|
| **GET** | Получение данных | ❌ Нет |
| **POST** | Создание нового ресурса | ✅ Да |
| **PUT** | Полное обновление ресурса | ✅ Да |
| **PATCH** | Частичное обновление ресурса | ✅ Да |
| **DELETE** | Удаление ресурса | ❌ Нет |

---

## 5. HTTP-статус-коды (самые популярные)

### Успешные (200–299)
| Код | Описание | Пример ситуации |
|-----|----------|-----------------|
| **200 OK** | Запрос выполнен успешно | Получены данные |
| **201 Created** | Ресурс создан успешно | Новый пользователь создан |

### Ошибки клиента (400–499)
| Код | Описание | Пример ситуации |
|-----|----------|-----------------|
| **400 Bad Request** | Неверный запрос | Обязательные поля не заполнены |
| **401 Unauthorized** | Не авторизован | Нет токена доступа |
| **403 Forbidden** | Доступ запрещён | Недостаточно прав |
| **404 Not Found** | Ресурс не найден | Пользователь не существует |

### Ошибки сервера (500–599)
| Код | Описание | Пример ситуации |
|-----|----------|-----------------|
| **500 Internal Server Error** | Внутренняя ошибка сервера | Ошибка подключения к БД |

## 6. Формат JSON

**JSON (JavaScript Object Notation)** — текстовый формат обмена данными.

### Структура:
- **Объект** — в фигурных скобках `{ }`
- **Массив** — в квадратных скобках `[ ]`
- **Ключи** — всегда строки в двойных кавычках
- **Значения** — могут быть строками, числами, булевыми (`true`/`false`), `null`, объектами, массивами

### Пример JSON-объекта:
```json
{
  "name": "Иван",              // строка
  "age": 30,                   // число
  "isActive": true,            // булево
  "address": null,             // null
  "hobbies": ["чтение", "спорт"], // массив
  "company": {                 // вложенный объект
    "name": "ООО Ромашка",
    "position": "Аналитик"
  }
}
```

### Особенности JSON:
- Все данные в кавычках или без, в зависимости от типа
- Массивы могут содержать объекты
- Комментарии в стандартном JSON **не допускаются**

---

## 7. Формат XML

**XML (eXtensible Markup Language)** — расширяемый язык разметки.

### Структура:
- Данные в **тегах** (открывающий и закрывающий)
- Теги могут содержать **атрибуты**
- Обязательно наличие закрывающего тега

### Пример XML:
```xml
<user>
  <name>Иван</name>
  <age>30</age>
  <hobbies>
    <hobby>чтение</hobby>
    <hobby>спорт</hobby>
  </hobbies>
</user>
```

### Где используется:
- Чаще в **SOAP**-сервисах
- В **REST**-сервисах чаще используют JSON
- XML **поддерживает комментарии** (`<!-- комментарий -->`)

---

## 8. JSON vs XML — сравнение

| Критерий | JSON | XML |
|----------|------|-----|
| **Структура** | Ключ-значение, массивы | Теги (дерево) |
| **Читаемость** | Более компактный, легче читается | Более громоздкий |
| **Комментарии** | ❌ Нет (кроме JSON5) | ✅ Есть |
| **Типы данных** | Строки, числа, булевы, null | Всё как строки |
| **Популярность** | ✅ Более популярен, особенно в REST | Чаще в SOAP, старых системах |

## 9. Шпаргалка по HTTP-запросам

```
МЕТОД /путь?параметры HTTP/версия
Заголовок1: значение
Заголовок2: значение

Тело запроса (для POST/PUT/PATCH)
```

### Примеры запросов:

**GET — получить список заявок:**
```
GET /applications?status=new HTTP/1.1
Host: api.agency.ru
Authorization: Bearer token123
```

**POST — создать заявку:**
```
POST /applications HTTP/1.1
Host: api.agency.ru
Content-Type: application/json

{
  "title": "Java-разработчик",
  "salary": 150000,
  "description": "Опыт от 3 лет"
}
```

## 10. Итоговые тезисы (шпаргалка)

1. **HTTP** — протокол для передачи данных в интернете
2. Структура запроса: **метод + путь + версия + заголовки + тело**
3. Основные методы: **GET** (чтение), **POST** (создание), **PUT/PATCH** (обновление), **DELETE** (удаление)
4. **200** — успех, **400** — ошибка клиента, **500** — ошибка сервера
5. **JSON** — компактный формат с типами данных, без комментариев
6. **XML** — более громоздкий, с комментариями, используется реже
7. Тело запроса используется в **POST, PUT, PATCH**, но не в **GET**
8. В REST чаще используют **JSON**, в SOAP — **XML**
